# MVP ページ設計書
# ── 守護存在診断サービス 最初の公開版

作成日：2026-06-15

---

## MVPの目的と優先順位

**目的：** 守護存在診断を公開し、ユーザーの反応を確認する
**成功指標：** SNSシェア数・PDF購入率・離脱ポイントの把握

**優先順位：**
1. 診断が完走できること
2. 結果に「納得感」と「もっと知りたい」感があること
3. PDF購入動線が自然につながること

---

## サイト全体のフロー

```
TOPページ
  ↓ 「無料で診断する」ボタン
診断ページ（生年月日入力）
  ↓ 「守護を呼び覚ます」ボタン
ローディング演出（3〜5秒）
  ↓
結果ページ（無料）
  ↓ 「詳細鑑定書を手に入れる」ボタン
PDF購入・決済ページ（外部）
  ↓
PDF鑑定書のダウンロード or メール受け取り
```

## 1. TOPページ構成

### ブロック1：ヒーロー（ファーストビュー）

```
┌─────────────────────────────────┐
│                                 │
│  [背景：和紙テクスチャ×墨絵の守護存在シルエット]  │
│                                 │
│  生まれた日に、守護は決まっていた。          │
│                                 │
│  神獣・神使・妖異・守護霊──              │
│  日本の霊的守護体系 12体から、            │
│  あなたを護る存在を診断します。            │
│                                 │
│       [ 無料で診断する → ]            │
│                                 │
│  ※ 生年月日を入力するだけ。登録不要。        │
│                                 │
└─────────────────────────────────┘
```

**設計意図：**
- キャッチコピーを最上位に置き、一瞬で世界観を伝える
- CTAは1つだけ。迷わせない
- 「登録不要」の一言で離脱を防ぐ

---

### ブロック2：3ステップ説明

```
STEP 1          STEP 2          STEP 3
生年月日を      守護存在が       結果と守護の
入力する   →   決まる      →   言葉を受け取る

所要時間：約1分
```

---

### ブロック3：守護存在の紹介（12体チラ見せ）

4階層の説明＋各3体のアイコン（名前のみ）を並べる

```
【神獣】    龍神  鳳凰  麒麟
【神使】    稲荷狐  八咫烏  白蛇
【妖異】    九尾の狐  猫又  天狗
【守護霊】  雪女  座敷童子  護り鬼
```

**設計意図：**
- 12体の名前を見て「私はどれだろう？」という期待感を作る
- 階層の説明は1〜2行程度にとどめる（詳細は結果ページで）
- クリックしても詳細には飛ばない（診断を先に完走させる）

### ブロック4：SNSシェア例（ソーシャルプルーフ）

```

 計算して動きすぎるとこある…刺さった」

「龍神×雪女の組み合わせ。
 "白夜の知者" なんか泣けた」
```

---

### ブロック5：再CTA

```

[ 無料で診断する → ]
```

## 4. 無料版で表示する内容（整理）

| 項目 | 表示 | 備考 |
|---|---|---|
| 主守護存在 名前 | ✅ | 大きく表示 |
| 主守護存在 二つ名 | ✅ | 名前の直下 |
| 主守護存在 階級 | ✅ | バッジ表示 |
| 主守護存在 守護属性（属性名） | ✅ | シンプルに |
| 主守護存在 才能の核 3つ | ✅ | 箇条書き |
| 主守護存在 得意な人生テーマ 2つ | ✅ | タグ形式 |
| 主守護存在 弱点 | ❌ | PDF |
| 主守護存在 開運行動 | ❌ | PDF |
| 副守護存在 名前・二つ名・階級 | ✅ | 小さく表示 |
| 副守護存在 才能・弱点・開運 | ❌ | PDF |
| 組み合わせタイトル | ✅ | 中身はなし |
| 組み合わせ読み（詳細） | ❌ | PDF |
| 守護存在からの一言（短文） | ✅ | ブランドボイス体験 |
| 2026年 年間メッセージ | ❌ | PDF |
| 月別アドバイス | ❌ | PDF |
| 相性の守護存在 | ❌ | PDF |
| SNSシェア機能 | ✅ | 必須 |

**無料の設計原則：**
- 「嬉しい・納得感がある」程度には届ける
- 「でもまだ知りたい」という余白を必ず残す
- 弱点と開運行動は必ずPDFに回す（最も「知りたい」情報）

## MVP実装の優先順位

### フェーズ1：動くものを作る

1. TOPページ（ヒーロー＋3ステップ＋CTA）
2. 診断ページ（生年月日入力フォーム）
3. ローディング演出
4. 結果ページ（全セクション）
5. SNSシェア機能

### フェーズ2：収益化

6. PDF鑑定書テンプレート（1組み合わせ分を試作）
7. 決済連携（Stripe or BASE等）
8. PDF配信（手動 or 自動）

### フェーズ3：コンテンツ拡充

9. 144通りの組み合わせテキスト執筆
10. イラスト・象徴紋の制作
11. SNSシェア画像（OGP）の制作

## 技術構成（MVP推奨）

| 要素 | 推奨 | 理由 |
|---|---|---|
| フレームワーク | Next.js（App Router） | SEO・OGP対応しやすい |
| スタイル | Tailwind CSS | 和風UIの実装が高速 |
| アニメーション | CSS + Framer Motion | ローディング演出 |
| ホスティング | Vercel | デプロイが最速 |
| 決済 | Stripe | シンプルな1商品決済 |
| PDF生成 | 手動（初期） → react-pdf（後期） | MVP段階は手動で十分 |
| 分析 | Google Analytics / Vercel Analytics | 離脱ポイントを計測 |

## 測定すべきKPI（公開後）

| KPI | 目標（目安） | 測定方法 |
|---|---|---|
| 診断完走率 | TOP→結果 70%以上 | GA イベント計測 |
